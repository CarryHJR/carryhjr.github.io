武大超算申请流程

Posted by Hu Jiarui on 2019-10-14

README

超算价格公道，校内访问非常稳定，非常推荐

超算中心简介

网址:http://hpc.whu.edu.cn
操作系统：x86_64架构的64位CentOS 7.5 Linux
GPU集群：400块Nvidia Tesla V100，峰值计算能力3000万亿次/秒，100G OPA互联

收费和缴费

官方文档:http://hpc.whu.edu.cn/info/1008/1057.htm
免费范围:
校内用户免费范围暂定为：每用户CPU 16核。

付费范围:
校内用户 0.06 元/核/h ; 1 元/卡/h

缴费：
每月初会收到费用统计邮件，到工学部信息中心203室开缴费单，然后校内转账

申请流程

老师和学生分别申请免费账户，通过后会收到邮件，邮件里提供了用户名、ip和初始密码，然后参照http://hpc.whu.edu.cn/sjfw/ffsq.htm 发送申请表到官方邮箱，所属超算用户填写老师的用户名，超算用户填写学生的用户名，申请通过后会收到邮件，记得加入官方qq群137281704

使用方式

基本概念

先理解两个概念:登录节点和运算节点。
登录节点就是ssh之后的节点，用hostname查看节点名，这个节点或者说这个主机只能用来基本操作如登录、编译文件等，不能用来进行大的运算，如果cpu占用率高的话会被管理员kill掉

运算节点分为cpu节点和gpu节点，武大超算进行了专门分区, CPU区: hpxg hpib pub GPU区: gpu，使用sinfo命令查看节点状态(idle表示空闲，drain表示不可用)

实际操作

武大超算采用slurm作业调度系统，有三种方式运行:
交互模式，以 srun 命令运行；(强烈推荐)
批处理模式，以 sbatch 命令运行，sbatch是要接特定格式的脚本，好处是自动将输出保存到指定文件，而且会在后台运行；
分配模式，以 salloc 命令运行，好处是可以分配固定时间的特定节点。

这里强烈推荐srun+tmux，完全满足日常需求。
